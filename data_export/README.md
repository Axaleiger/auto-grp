# Выгрузка данных → платформа ЦДП

## Зачем

Три сервиса (Закачка, Факторы обводнённости, Запускные параметры) работают от одного пакета данных.
Пакет собирается скриптом по `layer_id` / `field_id`, сохраняется как JSON, отправляется на почту и загружается в каталог платформы (**Данные**).

## Файлы

| Файл | Назначение |
|------|------------|
| `manifest_datasets.json` | Полный список необходимых датасетов и полей |
| `config.example.json` | Шаблон запроса (ID пластов, период, список датасетов) |
| `export_data.py` | Скрипт выгрузки |
| `sample_export.min.json` | Пример / шаблон готового JSON для платформы |
| `../platform_data.js` | Загрузка JSON в браузере (IndexedDB) |

Шаблоны также скачиваются из каталога платформы → **Данные**.

## Вид JSON для платформы

См. `sample_export.min.json`. Корневые поля:

- `schema`: `"cdp-platform-export"`
- `layers.<label>`: trunks, blocks, mor, tr, maps, grp_ports, startup
- `history` (опционально)
- `meta` / `request`: месторождение, период, `layer_id`

## Как собрать JSON

```bash
cd data_export
copy config.example.json config.json
# заполните layer_id, field_id, period
python export_data.py --config config.json
```

- `connection.mode: "local_csv"` — из папок пластов репозитория  
- `connection.mode: "corp_api"` — корпоративный API (токен `CDP_DATA_TOKEN`, допишите URL в `CorpApiClient`)

## Как загрузить в платформу

1. Откройте каталог → **Данные**
2. Скачайте шаблон при необходимости, заполните / подставьте выгрузку
3. Загрузите `cdp_export.json`
