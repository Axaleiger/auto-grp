# Подготовка данных для платформы ЦДП

## Зачем

Три сервиса (Закачка, Факторы обводнённости, Запускные параметры) работают от одного JSON-пакета.
Пакет собирается скриптом по `layer_id` / `field_id` и подключается в разделе **Данные**.

## Файлы

| Файл | Назначение |
|------|------------|
| `manifest_datasets.json` | Список необходимых датасетов и полей |
| `config.example.json` | Пример конфига запроса (ID пластов, период, датасеты) |
| `export_data.py` | Скрипт сборки JSON |
| `sample_export.min.json` | Пример структуры пакета |
| `../platform_data.js` | Подключение пакета в браузере (IndexedDB) |

Шаблоны доступны в каталоге платформы → **Данные**.

## Скрипт для IDE (основной)

Файл **`pull_cdp_data.py`** — единый скрипт выгрузки для IDE / Jupyter.

В начале файла расположен блок **«НАИМЕНОВАНИЯ ИЗ КЛАДА»**:

- `NAME_MER` — МЭР: скважина, дата, закачка / жидкость / нефть (помесячно)
- `NAME_TRUNKS`, `NAME_BLOCKS`, `NAME_TR`, `NAME_MAPS_PPL`, `NAME_GRP_PORTS` и др.
- для каждого блока приведено краткое описание состава данных
- необходимо запросить точные наименования сущностей в КЛАДе и указать их в переменных `NAME_*`
- пустое значение означает, что соответствующий блок не запрашивается

При отличии наименований колонок необходимо также заполнить `FIELD_WELL`, `FIELD_DATE` и др.

Далее:

1. Указать URL, токен, `layer_id` пластов и период выгрузки  
2. Выполнить скрипт — формируется `cdp_export.json`  
3. Подключить файл в каталоге платформы → **Данные**

Доступ из UI: каталог → **Данные** → «Скрипт Python (IDE)».

Дополнительно: `export_data.py` + `config.example.json` — расширенный вариант (в т.ч. local_csv).

## Структура JSON

См. `sample_export.min.json`. Корневые поля:

- `schema`: `"cdp-platform-export"`
- `layers.<label>`: trunks, blocks, mor, tr, maps, grp_ports, startup
- `history` (опционально)
- `meta` / `request`: месторождение, период, `layer_id`

## Расширенный запуск через config

```bash
cd data_export
copy config.example.json config.json
python export_data.py --config config.json
```

- `connection.mode: "local_csv"` — из папок пластов репозитория  
- `connection.mode: "corp_api"` — корпоративный API

## Подключение в платформе

Каталог → **Данные** → выбрать готовый JSON-файл.
